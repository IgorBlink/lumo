Lumo is a Turing-complete interpreted programming language where **every operator and keyword is a plain English word**. There are no symbols — no `+`, `>`, `==`, `{`, or `&&` — anywhere in program logic.

Lumo is designed to be written, read, and generated by large language models without ambiguity, while staying fully readable by humans.

```
define fibonacci taking n
  if n atmost 1 then
    return n
  end
  return call fibonacci passing n minus 1 plus call fibonacci passing n minus 2
end

print call fibonacci passing 10
```

---

## Why Lumo

Traditional programming languages use symbol clusters (`&&`, `||`, `!=`, `->`, `::`) optimised for expert human brevity. This creates friction for LLMs:

- Symbols tokenise inconsistently across tokenisers
- Visually similar operators (`=` vs `==` vs `===`) get confused under distribution
- Hallucinated syntax passes a surface read but fails at runtime

Lumo eliminates the symbol layer entirely. Every construct has **one canonical form**, **one meaning**, and **one token** — making LLM generation reliable and output verification trivial.

## Install

**Requirements:** C++17 compiler, CMake 3.10+

```bash
git clone https://github.com/IgorBlink/lumo.git
cd lumo
cmake -S . -B build
cmake --build build
./build/lumo examples/hello.lumo
```

Or build directly:

```bash
clang++ -std=c++17 -Iinclude src/*.cpp -o lumo
./lumo examples/hello.lumo
```

## Usage

```bash
./lumo <file.lumo>            # Run a program
./lumo --validate <file.lumo> # Check syntax only — exits 0 on OK
./lumo --ast <file.lumo>      # Print the AST and exit
./lumo                        # Read program from stdin
```

## Language at a Glance

| Concept | Lumo syntax |
|---|---|
| Declare variable | `let x be 10` |
| Mutate variable | `set x be x plus 1` |
| Arithmetic | `plus` `minus` `times` `divby` `modulo` |
| Comparison | `equals` `above` `below` `atleast` `atmost` |
| Logic | `and` `or` `not` |
| Conditional | `if … then` / `elif` / `else` / `end` |
| Pattern match | `match` / `when` / `catch error yield` / `end` |
| Count loop | `repeat 5 times` … `end` |
| While loop | `repeat while x above 0` … `end` |
| Iteration | `for each item in mylist` … `end` |
| Function | `define f taking x, y` … `return …` `end` |
| Call | `call f passing 1, 2` |
| List | `list 1, 2, 3` / `get items at 0` / `put items at 0 be 99` |
| Pipeline | `pipe name` / `start with` / `map` / `filter when` / `yield` |
| Print | `print "hello"` |
| Read input | `read name` |
| Annotation | `intent "description"` |
| Hole | `??? "not implemented yet"` |
| No-op | `skip` |

All blocks end with `end`. Indentation is cosmetic only.

## VS Code Extension

The [`vscode-extension/`](vscode-extension/) directory contains a VS Code extension providing:

- **Syntax highlighting** for all Lumo keywords, operators, strings, numbers, and comments
- **Real-time linting** — parse errors appear as red squiggles as you type, powered by `lumo --validate`
- **File icons** for `.lumo` files

### Install the extension

```bash
cd vscode-extension
npm install
npm run compile
npx @vscode/vsce package --no-dependencies --allow-missing-repository
```

Then in VS Code: `Cmd+Shift+P` → `Install from VSIX` → select the generated `.vsix` file.

Point the extension to your binary in VS Code settings:

```json
{
  "lumo.executablePath": "/path/to/lumo"
}
```
